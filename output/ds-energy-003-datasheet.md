## Data dictionary

Shape: tidy long format — **one row per `country` (location) per `year`**. Identifier columns plus a wide set of energy indicator columns. Field names, descriptions, units and sources below are verified against `owid-energy-codebook.csv` (retrieved 2026-06-28). The full dataset has on the order of ~130 indicator columns; a representative, verified subset is documented here, and the codebook is the authoritative complete list.

| Field | Type | Units | Description | Allowed values / null handling |
|---|---|---|---|---|
| `country` | string | — | Geographic location (country or region/aggregate). | Non-null key. Includes aggregates (e.g. "World", regions). |
| `year` | integer | year (CE) | Year of observation. | Non-null key. Range varies by indicator. |
| `iso_code` | string | — | ISO 3166-1 alpha-3 three-letter country code. | Null for aggregates/regions that have no ISO code. |
| `population` | number | people | Population by country (long historical series). | May be null where unavailable. |
| `gdp` | number | international-$ (2011 prices) | Total inflation-adjusted economic output (Maddison Project Database 2023). | May be null where unavailable. |
| `biofuel_consumption` | number | TWh | Primary energy consumption from biofuels. | Null where not reported. |
| `biofuel_cons_per_capita` | number | kWh | Biofuel consumption per capita. | Null where not reported. |

| `biofuel_cons_change_twh` | number | TWh | Annual change in biofuel consumption. | Null where prior-year base missing. |

- Units convention: absolute energy flows are typically in **TWh**; per-capita figures in **kWh**; shares in **%**; carbon intensity in **gCO₂/kWh**.
- Missing values are encoded as empty cells (blank/`NaN`) in the CSV rather than sentinel numbers.

## Datasheet for Datasets

- **Motivation:** Provide a single, tidy, regularly maintained global energy dataset so researchers, journalists, educators and policymakers can analyse energy consumption, production and the electricity mix over time without reconciling many disparate upstream sources. Compiled by Our World in Data.
- **Composition:** Instances are (country/region, year) observations. Each row carries identifiers (`country`, `year`, `iso_code`), context (`population`, `gdp`) and a wide set of energy indicators per fuel type. Includes country-level rows and aggregate rows (e.g. "World", regions). Contains **no personal or individual-level data** — all values are macro aggregates.
- **Collection process:** OWID ingests and harmonises third-party series — principally the Energy Institute Statistical Review of World Energy, Ember (Yearly Electricity Data), the Maddison Project Database (GDP), and OWID/UN population sources — then standardises units, country names and codes.
- **Preprocessing / cleaning:** Standardisation to consistent units (TWh, kWh, %), consistent country naming and ISO codes, computation of derived indicators (per-capita, shares, year-on-year changes). Processing code is published in the repository. Missing data is left blank rather than imputed.
- **Uses:** Trend analysis of energy transition, fuel mix, decarbonisation and per-capita consumption; teaching; journalism; input to models. Not a substitute for primary regulatory/financial energy reporting. Users should respect upstream source caveats and revision cycles.
- **Distribution:** Publicly distributed via the GitHub repository (`owid-energy-data.csv/.xlsx/.json`) under CC BY 4.0. This datasheet does not redistribute the data — only the upstream URL.
- **Maintenance:** Maintained by Our World in Data; updated as upstream sources release new editions (the codebook entries reflect source vintages through 2025–2026). Issues/contributions via the GitHub repository.

## Validation script

Documentation only — this script does **not** download, mirror, or host the dataset. Point it at a locally obtained copy (fetched by the user directly from the source under CC BY 4.0). It checks expected structure: required columns, key uniqueness, and a sanity row count.

```python
#!/usr/bin/env python3
"""Validate the structure of a local OWID energy-data CSV.

Usage:  python validate_energy.py path/to/owid-energy-data.csv
This performs read-only checks. It never downloads or republishes data.
"""
import sys
import csv

REQUIRED_COLUMNS = ["country", "year", "iso_code", "population"]
# Expect a substantial panel: many countries x many years.
MIN_EXPECTED_ROWS = 10000

def main(path: str) -> int:
    with open(path, newline="", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        header = reader.fieldnames or []

        missing = [c for c in REQUIRED_COLUMNS if c not in header]
        if missing:
            print(f"FAIL: missing required columns: {missing}")
            return 1

        seen_keys = set()
        rows = 0
        dup = 0
        for row in reader:
            rows += 1
            key = (row["country"], row["year"])
            if key in seen_keys:
                dup += 1
            seen_keys.add(key)

    print(f"columns: {len(header)}  rows: {rows}  duplicate (country,year) keys: {dup}")
    ok = True
    if rows < MIN_EXPECTED_ROWS:
        print(f"WARN: row count {rows} below expected minimum {MIN_EXPECTED_ROWS}")
        ok = False
    if dup:
        print(f"FAIL: {dup} duplicate (country, year) keys — expected unique panel keys")
        ok = False
    print("PASS" if ok else "CHECK FAILED")
    return 0 if ok else 1

if __name__ == "__main__":
    if len(sys.argv) != 2:
        print("usage: python validate_energy.py <local-csv-path>")
        raise SystemExit(2)
    raise SystemExit(main(sys.argv[1]))
```

## Limitations & gaps

- **Partial column coverage:** Only a representative, individually verified subset of columns is tabulated above. The dataset contains on the order of ~130 indicator columns; the per-fuel `<fuel>_*` families were documented by pattern, not row-by-row. The authoritative complete list is `owid-energy-codebook.csv`.
- **Row count / vintage not directly measured:** The `MIN_EXPECTED_ROWS` threshold and exact year ranges are heuristic; they were not confirmed by reading the full data file (documentation-only constraint — the dataset was not downloaded, sampled, or hosted).
- **Upstream licence nuance:** OWID's CC BY 4.0 covers OWID-produced compilations/code. Specific third-party source series may carry their own terms; verify upstream licences before reusing individual series commercially.
- **Source vintages drift:** Codebook source years (2023–2026) indicate ongoing updates; field availability and units can change between editions — re-verify against the live codebook at retrieval time.
- **No PII:** All records are country/region-year aggregates; no personal or individual-level data is present or documented.
